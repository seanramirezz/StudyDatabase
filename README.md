# Study Database Application

## 📌 Project Description
This project is a full relational database application designed to help students manage study sessions. It provides a web-based interface where users can create, view, and manage study-related data while storing all information in a PostgreSQL database.

The system demonstrates how a frontend interface interacts with a backend server and a relational database to perform real-world data operations such as inserting, updating, and retrieving records.

---

## 🚀 Features
- Add new study sessions
- View all stored sessions from the database
- Edit and update session information
- Delete sessions
- Real-time interaction between frontend and backend
- Persistent data storage using PostgreSQL

---

## 🛠️ Technologies Used
- Node.js
- Express.js
- PostgreSQL
- pgAdmin 4
- HTML
- CSS
- JavaScript

---

## 📂 Database Schema
The database is structured using relational tables to store and manage study session data.

Example tables include:
- **Users** – stores user information
- **StudySessions** – stores session details such as subject, time, and duration

Relationships:
- Each study session is linked to a user
- Primary and foreign keys are used to maintain data integrity
